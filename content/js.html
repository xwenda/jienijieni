Map和Set的区别，Map和Object的区别 ​

Set：

类似数组，但成员唯一，可用于数组、字符串去重。不会发生类型转换，使用的算法叫same-value-zero equality类似===，但是区别在于Set认为NaN===NaN，对象总是不相等

  • .size 返回成员总数

  • .add() 增加成员

  • .delete() 删除成员

  • .has() 查询成员

  • .clear() 清空成员 可用Array.from()将转化为数组

  • .keys() 返回键名

  • .values() 返回键值

  • .entries() 返回键值对，数组

  • .forEach() 回调函数遍历，没有返回值，只对数组结构处理默认遍历器生成函数就是values，所以for of可直接遍历

  • 其中Set没有键名，所以keys和values行为一致

  • 扩展运算符(...)内部使用的for of，故可用

  • 要改变原Set结构，需要用...重新赋值或者Array.from()中回调函数重新赋值

    WeakSet

    类似Set，不重复，成员只能是对象，对象都是弱引用，没有引用会自动回收，所以不适合引用，只适合临时存放对象 只有add、delete、has方法，没有size，不可遍历

Map：

类似对象，键名接受各种类型的值，每一个双元素的数组的数据结构都可以作为参数，Set和Map都可以生成新的Map，Map的键是和内存地址绑定的，解决了同名属性碰撞问题

  • size 返回成员

  • set(key,value) 设置新成员

  • get(key) 获取键值

  • has(key) 查询成员

  • delete(key) 删除成员

  • clear() 清空成员

  • .keys() 返回键名

  • .values() 返回键值

  • .entries() 返回键值对，数组

  • .forEach() 回调函数遍历，没有返回值，只对数组结构处理

  • 注意Map的遍历顺序是插入顺序

  • 默认遍历器生成函数就是entries

  • Map转为数组用...

    WeakMap

    只接受对象作为键名(null除外) 键名所指向的对象不计入垃圾回收机制，也是弱引用 WeakRef用于对象的弱引用 有一个方法deref()用于判断对象是否被回收

FinalizationRegistry：

用于制定目标对象被回收后执行的回调函数register()方法，用来注册所要观察的目标对象，取消已经注册的回调函数，则要向register()传入第三个参数作为标记，然后使用unregister(标记)取消

function debounce(fn,wait = 1000,immediate = false) {
	let timer = null
	return function debounced(...args) {
		if (timer) clearTimeout(timer)

		if(immediate && !timer) {
			fn.apply(this,...args)
			timer = setTimeout(() => {
				timer = null
			}, wait)
			return
		}

		timer = setTimeout(() => {
			fn.apply(this, ...args)
			timer = null
		}, wait)
	}
}

	return Object.prototype.toString.call(target).slice(8,-1)

function cloneDeep(target){
	let result
	let cloneTypeResult = cloneType(target)
	if(cloneTypeResult === 'Object'){
		result = {}
	} else if(cloneTypeResult === 'Array'){
		result = []
	} else{
		result = target
	}

	for(let i in target){
		let item = target[i]
		let itemType = cloneType(item)
		if(itemType === 'Object' || itemType === 'Array'){
			result[i] = cloneDeep(item)
		}else{
			result[i] = item
		}
	}
	return result
}

        this.failReason

            this.state = 'rejected'
            this.failReason = reason
            this.onRejectedCallbacks.forEach(fn => fn())
        }

        } catch (err) {
            reject(err)
        }
    }

Promise的all和race有什么区别 ​

all：

  • 接收一个Iterator结构的数据作为参数，其中的每个值都是promise实例
  • 只有状态都变成fulfilled才，all的状态才会变成fulfilled
  • 只有所有值中均没有reject，才会进入all的catch

race：

  • 和all相似，接收一个Iterator结构的数据作为参数，其中的每个值都是promise实例
  • 其中一个实例改变状态，race状态就跟着改变

堆和栈的区别 ​

栈(stack)：

  • 自动释放
  • 存储一些基础变量(string、number、boolean、null、undefined)以及一些对象的引用变量
  • 固定大小
  • 线性结构，后进先出
  • 读取速度快

堆(heap)：

  • 大小不固定
  • 不会自动释放，一般由程序员分配释放，也可以由垃圾回收机制回收
  • 存放function、array、object

instanceof的实现原理 ​

用于检测构造函数的 prototype属性是否是某个实例对象的原型链上

实现原理：不断遍历左边值的隐式属性 __proto__，判断其是否等于右边值的 prototype，直至等于 null为止

new的实现原理 ​

new

创建一个空对象obj 将obj的原型链上的__proto__指向构造函数的原型prototype 并且在原型链__proto__上设置构造函数的constructor为要实例化的fn 传入参数，并让构造函数的fn指向obj并执行 如果fn执行后返回的是对象类型(null除外)，则返回该对象，否则返回obj

function New(fn) {
	let obj = {}
	let arg = Array.prototype.slice.call(arguments,1)
	obj.__proto__ = fn.prototype
	obj.__proto__.constructor = fn
	let ret = fn.apply(obj,arg)
	return typeof ret === 'Object' ? ret || obj : obj
}

js运行机制和事件循环 ​

js其重要用途是用于交互，为了避免出现矛盾的交互情况，所以其只能是单线程语言，web worker允许js脚本创建线程，但子线程完全受主线程控制，且不得操作dom

js 的任务分为 同步任务和 异步任务，其中 异步任务进入 任务队列(task queue)

事件循环：

  1. 所有同步任务在主线程执行，形成一个执行栈
  2. 主线程外存在一个任务队列，异步任务有了结果，就在任务队列中放置一个事件
  3. 执行栈中的任务执行完成后，系统会读取任务队列中的事件，根据事件读取对应的异步任务，将对应的异步任务结束等待状态，进入执行栈，执行任务。
  4. 主线程不断执行第三步
  • 任务队列中的事件除了I/O还包括交互时间中制定了回调函数的事件
  • 任务队列是一个先进先出的结构，但是计时器需要先检查执行时间

nodejs的事件循环 ​

  • process.nextTick：将指定任务排到所有异步任务之前，运行效率高，不用检查任务队列
  • setImmedidate：将制定的任务排到下一次事件循环执行

onload 和 DOMContentLoaded的区别 ​

  • onload：在文档中的所有对象都在DOM中，所有图片，脚本，链接以及子框都完成了装载。
  • DOMContentLoaded：无需等待样式表、图像和子框架的完全加载

function getBlob(fileUrl) {
    return new Promise(resolve => {
        const xhr = new XMLHttpRequest();
        xhr.open('GET', fileUrl, true);
        // 监听进度事件，需要在请求发送前监听
        xhr.addEventListener(
            'progress',
            function (evt) {
                if (evt.lengthComputable) {
                    const percentComplete = evt.loaded / evt.total;
                    // percentage是当前下载进度，可根据自己的需求自行处理
                    const percentage = percentComplete * 100;
                    console.log(percentage)
                }
            },
            false
        );
        xhr.responseType = 'blob';
        xhr.onload = () => {
            if (xhr.status === 200) {
                resolve(xhr.response);
            }
        };
        xhr.send();
    });
}

    a.href = blobUrl
    document.body.appendChild(a);
    // 下载文件
    a.click();
    // 释放内存
    URL.revokeObjectURL(blobUrl);
    document.body.removeChild(a);
}

尾调用 ​

尾调用

即函数在最后一步调用函数自身

【function f() { return f() }】

必须是调用自身

注意，只有不再用到外层函数的内部变量，内层函数的调用帧才会取代外层函数的调用帧，否则就无法进行“尾调用优化”，若使用了内部变量，则形成了闭包

优化尾递归 ​

1、蹦床函数

js
function trampoline(f) {
  while (f && f instanceof Function) {
    f = f();
  }
  return f;
}

function sum(x, y) {
  if (y > 0) {
    return sum.bind(null, x + 1, y - 1);
  } else {
    return x;
  }
}

trampoline(sum(1, 100000))

2、真正的优化尾递归

js
function tco(f) {
  var value;
  var active = false;
  var accumulated = [];
  return function accumulator() {
    accumulated.push(arguments);
    if (!active) {
      active = true;
      while (accumulated.length) {
        value = f.apply(this, accumulated.shift());
      }
      active = false;
      return value;
    }
  };
}

var sum = tco(function(x, y) {
  if (y > 0) {
    return sum(x + 1, y - 1)
  }
  else {
    return x
  }
});

function hex2int(hex) {
    const len = hex.length
    let a = new Array(len), code;
    for (let i = 0; i < len; i++) {
        code = hex.charCodeAt(i);
        if (48 <= code && code < 58) {
            code -= 48;
        } else {
            code = (code & 0xdf) - 65 + 10;
        }
        a[i] = code;
    }
    return a.reduce((acc, c) => {
        acc = 16 * acc + c;
        return acc;
    }, 0);
}

function bfs(root) {
    const queue = [root];
