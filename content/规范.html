规范 ​

个人总结的代码规范

代码规范很重要，能够帮你很舒服的阅读代码，代码工整很容易维护，不容易出bug，出bug也容易找

但是规范是规范，并不一定需要要求的特别死，可根据自身阅读习惯自由调整

缩进 ​

如果公司有自定义的规则，则无视

否则规定两个空格作为缩进

选中代码片段tab、或者shift+tab控制缩进

js相关 ​

三元不等式 ​

js
aaa ? 1 : 2

在表达式较长时，推荐换行

js
xxlajsdlkjaklsd
  ? aslkdjasjdklasd(12308102938123)
  : lghldsfgjdfgdjklfgjdklfgkshdfgsdjlghdf(12837189072398)

当存在多个三元不等式嵌套时，一般要求换行

js
uiuqwoeui > 10 
  ? uiuqwoeui > 20
      ? 10
	  : 20
  : 30


uiuqwoeui <= 10 
  ? 30
  : uiuqwoeui > 20
	  ? 10
	  : 20

以上两种方式均可，重点是判断的方式

匿名函数 ​

js
(function(){}())
(function(){})()

必须在开头加;

js
;(function(){}())
;(function(){})()

关于变量的风格 ​

一般不超过编辑器的2/3的，均不推荐换行

js
const result = [
    'id',
    'name',
    'address',
    'old',
    'home',
]

const result = [ 'id', 'name', 'address', 'old', 'home', ]

以上均没严格要求，但是在变量不长的情况下，推荐一行，这样代码行数下去了，阅读会方便很多

类似

js
columns: [
  { title: '序号', scopedSlots: { customRender: 'serial' }, align: 'center', width: 50 },
  { title: '组件号', align: 'center', dataIndex: 'materialCode', width: 100 },
  { title: '任务包名称', dataIndex: 'taskPackageName', align: 'center', width: 200 },
]

至于每个变量的最后一项是否严格要求加,需要看公司项目是否要求，一般来说推荐加

if判断 ​

判断路径下的代码逻辑只有一句的情况下，可不用写{}

js
if (aaa) return 1

if (aaa) {
    return 1
}


if (aaa) return 1
else if (bbb) return 2
else return 3

if (aaa) {
    return 1
} else if (bbb) {
    return 2
} else {
    return 3
}

注意，仅仅是单句而且后续没有复杂判断的情况下，一般也可以写成这样，不过此种写法不推荐，进推荐在简短的代码中使用

js
return aaa && 1

vue相关 ​

template ​

  1. 必须严格要求缩进
  2. 一个组件元素只要长度不超过编辑器的2/3，可以写在一行，一旦超过必须换行，而且闭合标签必须和开头标签保持相同的缩进
vue
// 不推荐
<a-input 
  v-model="queryParam.MaterialCode" 
  disabled allcleared 
  readOnly 
  @click="1231" 
  @blur="123" />
vue
// 推荐
<a-input 
  v-model="queryParam.MaterialCode" 
  disabled allcleared 
  readOnly 
  @click="1231" 
  @blur="123" 
/>

import ​

  1. 没有用到的变量，必须手动删除，不要留在代码上
  2. import的对象属性，超过5个需要换行，注意严格保持缩进
html
// 不推荐
import { GetList, ImportExcel, GetList, ImportExcel, GetList, ImportExcel } from '@/api/statementCentre/electricalStatement'
html
// 推荐
import { 
  GetList,
  ImportExcel,
  GetList, 
  ImportExcel, 
  GetList,
  ImportExcel 
} from '@/api/statementCentre/electricalStatement'

name ​

必须为组件生命一个独立的name，此步骤不能缺，开头必须为大写字母

components ​

  1. 必须换行
  2. 结尾元素必须加,

props ​

  1. 在每一个属性不长时，推荐不换行，若其中有一项函数过长，则单独让它换行
  2. 最后一项必须加,
js
props: {
  visible: { type: Boolean, required: true },
  loading: { type: Boolean, default: () => false },
  model: { type: Object, default: () => null },
  formProperty: { type: Array, default: () => ['fileList', 'importDateTime'] },
},

data ​

  1. 变量声明的数据格式必须和其后续复制的数据格式一致

关于事件总线规范 ​

  1. 可在created/mounted等生命周期里面监听，但是必须在beforeDestroy注销监听

  2. 事件名称统一使用字符串，全局搜索也比较方便，并且不存在事件出错的问题，如下

prettier ​

背景 ​

  1. 前端代码格式本身松散
  2. 公司项目代码格式不一致
  3. 手动格式代码太过繁琐、心智负担过重
  4. 当前项目为多人开发，若大家都突然使用格式化插件，提交代码时容易造成大量冲突，见【过渡阶段注意事项】

阶段 ​

  1. 初始阶段-初次添加Prettier插件
  2. 过渡阶段-项目中存在【已格式化】/【未格式化】两种代码文件
  3. 完成阶段-项目中所有代码文件格式统一
  4. 进阶阶段-添加husky hooks在代码提交端格式化-优先级低
  5. 使用到的插件，Prettier，用于格式化代码

过渡阶段注意事项 ​

  1. 过渡阶段只使用【快捷键】格式化代码
  2. 在格式化某个【从未格式化】过的代码文件前，先咨询组内成员该文件是否有人正在更新，若有成员正在编辑此文件，可等对方编辑完，合并代码后再进行格式化，若短时间该成员不能结束编辑，则放弃此次格式化

安装插件及其配置 ​

  1. 安装prettier插件npm install --save-dev prettier@3.2.5
  2. 添加/.prettierrc.json配置文件，前端-常见的Prettier配置
JSON
{
  "printWidth": 150,
  "tabWidth": 2,
  "useTabs": false,
  "semi": true,
  "singleQuote": true,
  "proseWrap": "preserve",
  "arrowParens": "always",
  "bracketSpacing": true,
  "endOfLine": "lf",
  "eslintIntegration": false,
  "htmlWhitespaceSensitivity": "ignore",
  "ignorePath": ".prettier.ignore",
  "jsxBracketSameLine": false,
  "jsxSingleQuote": false,
  "requireConfig": false,
  "stylelintIntegration": false,
  "trailingComma": "none",
  "tslintIntegration": false
}
  1. 添加忽略文件/.prettier.ignore
Plain
public/
  1. Vue的编译器在开发环境默认会对安装Prettier的项目进行检查，由于项目许多文件处于未格式化的状态，所以要禁用该功能，vue-loader配置文档
JavaScript
// vue.config.js
module.exports = {
  chainWebpack: config => {
    config.module
      .rule('vue')
      .use('vue-loader')
      .tap(options => {
        options.prettify = false;
        return options;
      });
  }
};

编辑器Webstorm配置 ​

  1. 完成【安装插件及其配置】
  2. 进入【设置/setting】->【语言&框架/Language&Frameworks】->【JavaScript】->【Prettier】（webstorm内置prettier）
  3. 选择手动配置，将Prettier指向当前目录下的插件
  4. 将你想要格式化的文件后缀配置在【Run for files】
  5. 选中【Run on 'Reformat Code' action Ctrl+Alt+L】
  6. 注意：过渡阶段不要选中Run on save
  7. 点击Apply，可以使用 Ctrl+Alt+L 来格式化代码了
  8. 若执行格式化无响应，可能是未识别到.prettierrc文件，可重启，或者进入.prettierrc文件按，顶部会提示是否使用此文件进行prettier配置，点击【yes】即可

编辑器VSCode配置 ​

  1. 完成【安装插件及其配置】
  2. 安装VSCode【Prettier】插件
  3. 进入设置页配置*setting.json*
JavaScript
{
    "prettier.configPath": "./.prettierrc",

  1. 注意：需要去除【自动保存格式化配置】
  2. 以上配置完成，则可以使用【Shift + Alt + F】进行格式化，具体【快捷键】可右击文件查看
